Homework3 - Serialization

Метод saveObjectToFile - Генерирует уникальное имя файла на основе имени класса и UUID. Открывает ObjectOutputStream для записи объекта в файл. 
Записывает объект в файл. Закрывает поток вывода. В случае ошибки выводит сообщение об ошибке.

Метод loadObjectFromFileAndDelete - Открывает ObjectInputStream для чтения объекта из файла. Читает объект из файла. 
Закрывает поток ввода. В случае ошибки выводит сообщение об ошибке. Удаляет файл после успешной загрузки объекта. 
Если удаление файла не удалось, выводит сообщение об ошибке.

В качестве примера, создаем объект для сохранения (строка "Hello, Serialization!"). Сохраняем  его в файл с использованием saveObjectToFile.
Затем загружаем его из файла и удаляем с использованием loadObjectFromFileAndDelete. В случае успеха выводится сообщение с его содержимым.

В случае отсутствия файла: в методе loadObjectFromFileAndDelete будет перехвачено исключение FileNotFoundException при попытке открыть поток для чтения из файла, который не существует. В этом случае будет выведено сообщение об ошибке, но программа продолжит выполнение без попыток чтения или удаления файла.

В случае наличия файла с некорректными данными: Если файл существует, но его содержимое не является сериализованным объектом 
(например, если кто-то изменил файл вручную), возникнет исключение ClassNotFoundException или IOException в методе loadObjectFromFileAndDelete. 
Также будет выведено сообщение об ошибке, и программа продолжит выполнение без попыток загрузки или удаления файла.

В обоих случаях программа обрабатывает ошибки, выводит соответствующие сообщения об ошибке, и затем продолжает выполнение без прерывания.
