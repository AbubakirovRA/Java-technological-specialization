класс MethodExecutionAspect является аспектом, 
который определяет совет "вокруг" (around advice) выполнения метода с аннотацией @RecoverException. 
Этот аспект обрабатывает исключения, возникающие во время выполнения метода, 
и возвращает значение по умолчанию в случае исключения, 
если оно не указано в списке исключений для восстановления (noRecoverFor).

В методе handleException аспекта MethodExecutionAspect используется аннотация @Around, 
чтобы определить совет вокруг (around advice), который будет применяться к методам с аннотацией @RecoverException. 
Этот совет оборачивает выполнение метода и обрабатывает исключения в случае их возникновения.

Методы isExceptionInHierarchy и getDefaultValueForReturnType служат вспомогательными методами 
для проверки типа исключения и получения значения по умолчанию для типа возвращаемого значения соответственно.

Таким образом, этот файл содержит аспект, 
который определяет логику обработки исключений и возвращения значений по умолчанию для методов с аннотацией @RecoverException.

В примере контроллер ExampleController имеет единственный метод exampleEndpoint(),
который обрабатывает GET-запросы по пути /example. 
При вызове этого метода будет вызван метод methodWithTimerAnnotation() из ExampleService, 
что приведет к задержке в 2 секунды.

Чтобы сделать запрос к приложению и увидеть задержку, можно использовать инструмент для отправки HTTP-запросов, 
типа curl, Postman или Insomnia, или просто открыть браузер и ввести URL-адрес эндпоинта в адресной строке.

Например, отправить GET-запрос на http://localhost:8080/example. 
После отправки запроса увидим задержку, а затем получим ответ "Example response".
